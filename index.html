Vision

  1. User Experience
    • Follow the leaders (Google, Twitter, etc.) - reduced training, more intuitive, less resource intensive
    • End user UI management
    • Guide the user through the processing
    • Tell the user not only that there is an error, but why and if possible, how to fix
  2. Architecture
    • Built to a standard - not a browser
    • Responsive layout - not based on a screen size
    • No "business logic" in presentation layer
    • No batch reporting
    • Decouple as much as possible from server HW/SW changes
    • Lighten load on network
  3. Development
    • Short learning curve for 80% - 90% of presentation layer work
    • Use off the shelf .js libraries
    • Use off the shelf .css libraries
    • Modular components where possible
    • Use page templates so focus can be on business layer and UI components directly tied to that data
    • As there is one source of truth for data, code will only be written once
    • This becomes the "how to" source
  4. Accept that we will struggle at first because this is so different than our typical web development
